# fix-3c44c4bdcf459972 — E2T2/E2T3 'Close poll' button uses wrong callback format — silently broken

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

E2T2.ts:26 (`buildLivePollKeyboard`) and E2T3.ts:163 (back-to-poll keyboard) construct a "Close poll" button with callback data `poll:close:<pollId>` — only the poll ID.

E3T1.ts:37-49 (the actual close handler) expects format `poll:close:<creatorUserId>:<messageId>:<pollId>` (5 colon-separated parts). When `parts.length < 5`, E3T1.ts:42 responds with an empty `answerCallbackQuery()` and returns — the button does nothing visible to the user.

When E2T2's live-results edit replaces the poll message's keyboard, it overwrites E1T4's correctly-formatted close button (`poll:close:<creatorUserId>:<messageId>:<pollId>` from E1T4.ts:61) with the broken format. The "Close poll" button becomes non-functional for any poll whose message has been edited by E2T2.

**Fix:** `buildLivePollKeyboard` and the back-to-poll keyboard must pass `creatorUserId` and `messageId` alongside `pollId`, using the same format as E1T4.ts:61.
