# fix-263f40bae8008617 — E6T5.json: Spec does not test closed poll behavior (tests non-existent polls instead)

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

The task requires "BotSpec tests for closed poll behavior", but all 3 test cases in `E6T5.json` lack KV setup data. Without seeded poll records, the close flow behaves as follows:
1. `poll:close` silently succeeds (E3T1 calls `pollStore.close()` which returns `null` when the poll doesn't exist, and E3T1 ignores the return value)
2. `poll:results` returns "Poll not found" — the poll was never actually created or closed
3. `poll:export` returns "Poll not found" — same reason

The tests verify **non-existent poll handling**, not **closed poll behavior** as the task requires. A proper closed poll test needs KV setup with a real poll record plus votes, then closes it and verifies results/export work on the closed (but extant) poll.

**Fix:** Add `setup.kv` and `setup.sets` entries to seed a poll with options and votes, then test close/results/export on the **existing**, now-closed poll.
