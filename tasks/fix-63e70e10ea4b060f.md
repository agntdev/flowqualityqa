# fix-63e70e10ea4b060f — E2T3 resolveVoterName makes N sequential getChatMember calls without batching

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

E2T3.ts:88-91 iterates over up to `MAX_VOTERS_SHOWN` (10) voters in a for loop, calling `resolveVoterName()` sequentially for each. Each call invokes `ctx.api.getChatMember(chatId, userId)` (E2T3.ts:24), a Telegram API request. For a named poll with 10 voters, this means 10 sequential network round-trips — making the Results display noticeably slow (potentially multiple seconds) and risking Telegram rate limits.

**Fix:** Collect all user IDs first, then either batch them into a single API call (not directly possible with `getChatMember`), or fetch them concurrently with `Promise.all` to reduce wall-clock time. Alternatively, cache chat member names locally with a reasonable TTL.
