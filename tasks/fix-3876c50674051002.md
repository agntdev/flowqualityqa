# fix-3876c50674051002 — Vote changes silently ignored (E4T3)

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

When a user changes their vote in a non-anonymous poll, the handler in `src/handlers/E4T3.ts` calls `store.getVoteByUserAndPoll()` which finds the existing vote and executes `continue`, skipping creation of the new vote. The old vote persists forever; vote changes are silently dropped. The handler should either delete the old vote record (and its user-poll marker) before creating the new one, or add an `update` method to `VoteStore` that replaces the existing vote in-place.

This contradicts the expected behavior of a poll system where vote changes should be reflected. The one-vote-per-user-per-poll constraint should mean _at most one at a time_, not _first vote wins forever_.
