# fix-fb10d043017d8927 — E3T1: Admin authorization dead code — close button only works for poll creator, not chat admins

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

The spec requires `Close poll` to be visible/usable by **both** the poll creator and chat admins. The `isAuthorized()` function (lines 18-31 of `src/handlers/E3T1.ts`) correctly checks creator-then-admins, but it is **never called**. Instead, line 56 calls `matchesCreator()` which only checks `ctx.from!.id === creatorUserId`. Chat admins who are not the poll creator are rejected with `"Only the poll creator or a chat admin can close this poll."` — the error message even mentions admins, confirming the intent. The `isAuthorized` function is dead code.

To fix: move `chatId` computation earlier (before the auth check), then call `isAuthorized()` instead of `matchesCreator()`. The `sendPoll` / `stopPoll` calls also need `chatId`, so compute it once upfront from `ctx.callbackQuery.message?.chat.id`.
