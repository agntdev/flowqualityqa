# fix-7129c5a31b68b9de — E6T6.ts: Empty voter ID in public CSV passes validation as voter 0

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 FQABOT

In `validatePublicCsv()` (`src/handlers/E6T6.ts:104-107`), an empty voter field is parsed as `Number("")` which evaluates to `0`. Since `Number.isInteger(0) && 0 >= 0` is `true`, a CSV row like `Option,1,` (empty voter ID) passes validation with voter ID `0` instead of being rejected. This means the validator would accept CSV rows with missing voter data, potentially allowing invalid exports to pass validation.

**Fix:** Add a check that `fields[2]` is non-empty before passing it to `Number()`, e.g.:
```ts
if (!fields[2]) {
  issues.push(`Row ${i}: voter ID is missing`);
  continue;
}
```
