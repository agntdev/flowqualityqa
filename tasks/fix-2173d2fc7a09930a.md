# fix-2173d2fc7a09930a — E6T7.ts: Unreachable permission handlers — dead code

E6T7 implements `perm:close:` and `perm:export:` callback handlers with authorization logic (creator check + `getChatAdministrators`). However, **no UI button anywhere in the bot emits these callbacks**. The actual close/export buttons wired in the poll preview flow use:
- Close: `poll:close:<userId>:<msgId>:<pollId>` — handled by `E3T1.ts` (with its own authorization at line 58-71)
- Export: `poll:export:<pollId>` — handled by `E3T2.ts` (with its own authorization at line 52-59)

The `perm:close:` and `perm:export:` handlers can never be triggered by any real user action. The spec tests them in isolation, but no production code path reaches them. This is dead code.

**Fix:** Either wire the `perm:close:`/`perm:export:` callbacks into the actual UI flow (e.g., make E3T1/E3T2 delegate to them), or remove the handlers and use E3T1/E3T2's existing authorization directly.
